## なぜ仮想環境を使用すべきか

仮想環境は全ての Python プロジェクトの依存関係を他のプロジェクトの依存関係から分離します。これには以下のような利点があります。

1. 依存関係の管理が簡単になる
1. 新しい環境をすばやく起動し、異なるバージョンのライブラリとの互換性を検証することで、異なるバージョンのライブラリを使用、テストできるようにします。

## (必須)Python バージョン要件

このガイドは Python3.6 及 Python3.7 でテストされています。現時点では Python3.8 はサポートされていません。

## Mac OS X セットアップ

1. 仮想環境を配置するフォルダを作成します。`$ mkdir ~/python-envs`
1. `sample-env`という名前の仮想環境を作成するために次のコマンドを実行します。`$ python3 -m venv ~/python-envs/sample-env`
1. 作成した仮想環境をアクティブ化するために次のコマンドを実行します。`$ source ~/python-envs/sample-env/bin/activate`
1. pip のバージョンを最新にします。`$ pip3 install --upgrade pip`
1. setuptools のバージョンを最新にします。`$ pip3 install --upgrade setuptools`
1. 仮想環境を非アクティブ化するためには`$ deactivate`を実行します。再度アクティブ化するには手順 3 と同様に`activate`コマンドを実行します。
